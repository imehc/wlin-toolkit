## 功能说明

`digitUppercase` 函数将阿拉伯数字转换为标准的中文大写格式，适用于财务、法律等正式文档。支持整数和小数（精确到分）。

<DigitUppercase client:load />

## API 参考

```typescript
function digitUppercase(n: number): string
```

### 参数

- **n**: 要转换的数字（支持整数和小数，精确到分）

### 返回值

返回中文大写格式的字符串

### 限制

- 不支持负数
- 最大值不超过 1000 亿
- 小数位精确到分（两位）

### 示例代码

```javascript
import { digitUppercase } from 'wlin-toolkit';

// 整数转换
digitUppercase(0);
// "零元整"

digitUppercase(100);
// "壹佰元整"

digitUppercase(10011);
// "壹万零壹拾壹元整"

// 小数转换
digitUppercase(88.79);
// "捌拾捌元柒角玖分"

digitUppercase(123.45);
// "壹佰贰拾叁元肆角伍分"

// 大额数字
digitUppercase(12754894306.23);
// "壹佰贰拾柒亿伍仟肆佰捌拾玖万肆仟叁佰零陆元贰角叁分"
```

## 技术实现

此函数使用 Rust 编写并编译为 WebAssembly，采用高效的算法处理数字转换：

- **精确计算**: 使用 Rust 的精确数值类型，避免浮点数精度问题
- **规则引擎**: 实现完整的中文大写转换规则
- **错误处理**: 严格的输入验证，确保输出的准确性
- **高性能**: WebAssembly 执行速度远超 JavaScript 实现

## 常见问题

### 为什么小数只支持两位？

中文大写金额通常用于货币场景，货币的最小单位是分（0.01 元），因此只需支持两位小数。

### 如何处理超出范围的数字？

函数会抛出错误。建议在调用前验证输入：

```javascript
try {
  const result = digitUppercase(12345.67);
  console.log(result);
} catch (error) {
  console.error("转换失败:", error.message);
}
```

### 零的处理规则是什么？

函数严格遵循中文大写金额的规范：

- 数字中间的零需要读出
- 连续多个零只读一个
- 末尾的零不读
- 单独的 0 输出为"零元整"

示例：
- 10011 → "壹万零壹拾壹元整"（中间的两个零读为一个零）
- 10001 → "壹万零壹元整"（中间的三个零读为一个零）
